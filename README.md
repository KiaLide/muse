## ✨ 功能介紹

| 功能 | 說明 |
|------|------|
| 📥 **新增歌曲** | 貼上 YouTube、Spotify 連結或上傳本地音檔 |
| 📊 **品味分析** | 從你的歌單萃取 11 個音樂維度的偏好側寫 |
| 🔍 **智慧推薦** | 依品味搜尋 YouTube，套用風格／情緒／年代等多層篩選後推薦 |
| ⭐ **歌曲評分** | 對任意連結評分，顯示各維度與你偏好的對比 |

## 🚀 快速開始

### 方法一：直接下載（推薦）

1. 點選右上角 **Code → Download ZIP**，解壓縮到任意資料夾
2. 雙擊 **啟動.bat**
3. 第一次啟動會自動安裝 Python 和 FFmpeg（約 3–10 分鐘）
4. 安裝完成後瀏覽器會自動開啟 `http://127.0.0.1:5000`

### 方法二：Git Clone

```bash
git clone https://github.com/YOUR_USERNAME/muse.git
cd muse
# 雙擊 啟動.bat，或手動執行：
pip install -r requirements.txt
python server.py
```

### 手動安裝相依套件

```bash
pip install flask yt-dlp librosa numpy scipy soundfile
```

## 📖 使用說明

### 新增歌曲

在「**新增歌曲**」分頁貼上連結或上傳檔案：

| 支援格式 | 範例 |
|----------|------|
| YouTube 單曲 | `https://www.youtube.com/watch?v=...` |
| YouTube 播放清單 | `https://www.youtube.com/playlist?list=...` |
| Spotify 單曲 | `https://open.spotify.com/track/...` |
| Spotify 播放清單 | `https://open.spotify.com/playlist/...` |
| 本地音檔 | `.mp3` `.wav` `.flac` `.m4a` `.aac` `.ogg` |

**流程：**
1. 貼上連結 → 系統抓取歌曲資訊（標題、縮圖）
2. 確認後開始下載音訊（僅前 2 分鐘用於分析）
3. 萃取 83 維音訊特徵並存入資料庫
4. **原始音檔自動刪除**，僅保留特徵數值

> 💡 Spotify 連結：系統會讀取曲名和演出者，在 YouTube 上找到對應影片後分析

---

### 分析品味

切換到「**品味分析**」分頁，點選「**分析我的音樂品味**」：

- 系統計算你歌單中所有歌曲 83 維特徵的**平均值**
- 換算成 11 個直覺維度顯示（見下方說明）
- 以雷達圖、長條圖呈現你的音樂性格

---

### 推薦歌曲

在「**品味分析**」分頁下方，可設定推薦條件：

| 篩選項目 | 選項 |
|----------|------|
| 風格 | 電子、搖滾、金屬、爵士、古典、民謠、嘻哈、R&B、獨立、城市流行、舞曲、鄉村、藍調、龐克、雷鬼、放克、靈魂樂、流行、純音樂 |
| 情緒 | 快樂、平靜、活力、悲傷、浪漫、懷舊、振奮、夢幻、孤獨、夜晚 |
| 年代 | 60s–00s、2010s、2015s、2018s、2020s、2022s、2023s、2024s |
| 聲線 | 男聲、女聲 |
| 語言 | 中文、日文、韓文、英文、粵語 |

點選「**開始推薦**」後，系統會自動：
1. 依條件搜尋 YouTube
2. 對每個候選曲套用多層篩選（語言、年代、風格、情緒、聲線）
3. 計算與你品味的相似度
4. 依分數排序後顯示推薦清單

每張推薦卡片右下角有「**詳細分析**」按鈕，可查看此曲在各維度與你偏好的差距。

---

### 評分歌曲

切換到「**評分歌曲**」分頁，貼上 YouTube 連結：

- 系統分析此曲並計算與你品味的**整體相符度（0–100 分）**
- 顯示 11 個維度的對比：
  - 🟢 **匹配**：差距 < 15 分
  - 🔴 **偏高**：此曲數值比你的偏好高 15 分以上
  - 🔵 **偏低**：此曲數值比你的偏好低 15 分以上

---

### 設定檔管理

點選右上角的設定檔按鈕（顯示目前設定檔名稱）：

- **切換設定檔**：點選清單中的設定檔即可切換，歌單和品味完全獨立
- **重新命名**：雙擊設定檔名稱，輸入新名稱後按 Enter

適合用途：白天上班的音樂 / 睡前助眠的音樂 / 運動時的音樂

## 🔬 運作原理

### 音訊特徵分析

每首歌曲分析 **83 個原始特徵**，再換算成 **11 個直覺維度**：

| 維度 | 對應原始特徵 | 代表意義 |
|------|------------|---------|
| 🥁 節奏感 | BPM、拍點強度、節拍規律性 | 曲子有沒有明確的律動 |
| ⚡ 能量感 | RMS 音量、動態範圍 | 整體音量大小與爆發力 |
| 🎻 和諧度 | 諧波比例、色調對比 | 旋律性 vs 噪音性 |
| 😊 情感色彩 | 大調/小調色彩（Chroma） | 偏向明朗還是憂鬱 |
| 🎛️ 音色複雜度 | 頻譜頻寬、MFCC 離散程度 | 樂器層次是否豐富 |
| 🎤 人聲比重 | F0 偵測、諧波結構 | 有無人聲、人聲佔比 |
| 🔊 音場亮度 | 頻譜重心（Spectral Centroid） | 高頻多（清亮）vs 低頻多（厚重） |
| 💥 打擊感 | 打擊波比例、Onset 密度 | 鼓聲節奏感強弱 |
| 🌊 低頻厚度 | 低頻 MFCC 係數 | 貝斯、低音的比重 |
| 🎼 調性穩定度 | Tonnetz 穩定性、和聲張力 | 調性是否跳躍、複雜 |
| 🌀 音場密度 | ZCR（過零率）、頻譜通量 | 聲音的疏密、空曠感 |

> **分析工具：** [librosa](https://librosa.org/) — 開源音訊分析函式庫

---

### 相似度計算

```
相似度 = 餘弦相似度(你的特徵向量, 候選曲特徵向量)
```

- 你的「特徵向量」= 歌單中所有歌曲 83 維特徵的**平均值**
- 餘弦相似度只看**方向**，不受音量絕對值影響
- 範圍 0–1，越接近 1 代表風格越相近

---

### 推薦過濾流程

```
YouTube 搜尋結果（候選清單）
        │
        ▼ Layer 1：語言偵測（Metadata + Unicode 分析）
        ▼ Layer 2：重複過濾（已在歌單中則跳過）
        ▼ Layer 3：年代篩選（Metadata 年份 + 標題年份 + 網路搜尋）
        ▼ Layer 4：風格篩選 — Metadata 關鍵字（免下載）
        ▼ Layer 4b：情緒篩選 — Metadata 關鍵字（免下載）
        ▼ Layer 5：排除關鍵字（cover、remix、翻唱...）
        │
        ▼ 【下載音訊前 2 分鐘】（分析後立即刪除）
        │
        ▼ Layer 6：音訊特徵萃取 + 相似度計算
        ▼ Layer 7：風格 — 音訊信心值（搭配 Metadata 綜合判斷）
        ▼ Layer 8：情緒 — Metadata 確認 / 音訊輔助判斷
        ▼ Layer 9：聲線偵測（F0 基頻分析 + 網路搜尋）
        ▼ Layer 10：包含關鍵字確認
        │
        ▼ 計算相似度分數 → 排序 → 顯示推薦
```

**為什麼風格/情緒用 Metadata 為主？**

純音訊分析對「風格」和「情緒」的判斷誤差很大——例如輕搖滾和流行的頻譜很相似。所以系統**優先讀取 YouTube metadata（標題、描述、分類標籤）中的關鍵字**，只有在 metadata 沒有足夠資訊時才用音訊分析做輔助判斷。

---

### 評分邏輯

```
整體相符度 = 100 − 平均差距

其中：平均差距 = 11 個維度差距的平均值
      差距 = |此曲在某維度的數值 − 你的偏好均值|
```

範例：
| 維度 | 你的偏好 | 此曲 | 差距 | 判定 |
|------|---------|------|------|------|
| 節奏感 | 72 | 75 | 3 | 🟢 匹配 |
| 能量感 | 45 | 80 | 35 | 🔴 偏高 |
| 和諧度 | 60 | 55 | 5 | 🟢 匹配 |
| ... | ... | ... | ... | ... |

## 🏗️ 技術架構

```
muse/
├── server.py          # Flask 後端 — 所有 API、音訊分析、推薦邏輯
├── muse.html          # 單頁前端（純 HTML + CSS + JavaScript，無框架）
├── requirements.txt   # Python 相依套件
├── 啟動.bat           # Windows 一鍵啟動腳本（自動安裝 Python 和 FFmpeg）
└── muse.db            # SQLite 資料庫（本地，不上傳 GitHub）
```

### 後端（server.py）

| 技術 | 用途 |
|------|------|
| **Flask** | HTTP 伺服器、API 路由 |
| **yt-dlp** | YouTube 影片資訊擷取、音訊下載 |
| **librosa** | 音訊分析（MFCC、頻譜、節拍等） |
| **numpy / scipy** | 特徵向量計算、餘弦相似度 |
| **SQLite** | 本地資料庫（歌曲特徵、設定檔） |
| **Server-Sent Events** | 即時推送分析進度到前端 |

### 前端（muse.html）

- 純原生 HTML/CSS/JavaScript，無任何框架
- 單一 HTML 檔案，無需建置工具
- 透過 SSE（EventSource）接收後端即時進度

### 資料流

```
使用者貼上連結
    │
    ▼
Flask API 收到請求
    │
    ├─ yt-dlp 取得影片資訊（標題、縮圖、metadata）
    │
    ├─ yt-dlp 下載音訊（mp3，僅前 2 分鐘）
    │
    ├─ librosa 分析音訊 → 83 個原始特徵
    │
    ├─ 特徵存入 SQLite
    │
    └─ 刪除音訊暫存檔
```

## ❓ 常見問題

**Q: 第一次啟動很慢？**
A: 啟動.bat 需要下載安裝 Python（~25 MB）和 FFmpeg（~80 MB），以及 Python 套件（~300 MB）。安裝完成後之後每次啟動只需幾秒。

**Q: Spotify 連結可以用嗎？**
A: 可以。系統讀取曲名和演出者後，到 YouTube 搜尋對應影片來下載和分析。由於 Spotify 的 DRM 保護，音訊是從 YouTube 抓取，而非直接從 Spotify 下載。

**Q: 分析完的音檔會保留嗎？**
A: 不會。每首歌的音訊只下載前 2 分鐘用於分析，分析完成後立即刪除，只保留特徵數值（幾百 bytes）。

**Q: 我的歌單資料存在哪？**
A: 存在專案資料夾內的 `muse.db`（SQLite 資料庫）。這個檔案不會上傳到 GitHub（已加入 .gitignore）。

**Q: 推薦的相似度多高才算好？**
A: 通常 > 0.85 就非常接近你的風格，0.75–0.85 是合理範圍。低於 0.70 可能聽起來有些差異。

**Q: 可以在 Mac / Linux 使用嗎？**
A: server.py 和 muse.html 本身跨平台，但 啟動.bat 僅限 Windows。Mac/Linux 用戶請手動執行：`pip install -r requirements.txt && python server.py`
